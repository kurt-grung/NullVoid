# NullVoid v2.1.0 Release Notes

**Release Date:** January 27, 2025  
**Version:** 2.1.0  
**Type:** Minor Release - IoC Integration & Performance Optimizations

## üéØ Overview

NullVoid v2.1.0 introduces groundbreaking features for enterprise security scanning with Public IoC (Indicators of Compromise) Integration and comprehensive performance optimizations. This release transforms NullVoid into a complete security scanning solution with vulnerability database integration, intelligent caching, and network optimizations.

## üöÄ Key Features

### üõ°Ô∏è **Public IoC Integration**

NullVoid now integrates with multiple vulnerability databases to check packages for known security issues:

- **Snyk Provider**: Commercial vulnerability database (requires API key)
- **npm Advisories**: npm's official security advisories (public, no auth required)
- **GitHub Security Advisories (GHSA)**: GitHub's security database (public, enhanced rate limits with token)
- **NVD/CVE**: National Vulnerability Database (public, optional API key for higher rate limits)

#### Features:
- **Automatic Vulnerability Detection**: Scans all dependencies in `package.json` automatically
- **Multi-Provider Aggregation**: Combines results from multiple sources for comprehensive coverage
- **Intelligent Deduplication**: Removes duplicate vulnerabilities across providers
- **Rate Limiting**: Built-in rate limiting per provider to respect API limits
- **Caching**: 1-hour cache for vulnerability data to reduce API calls
- **Provider Priority**: Configurable provider priority for result aggregation

#### Usage:
```bash
# Enable IoC scanning (default)
nullvoid /path/to/project

# Select specific providers
nullvoid /path/to/project --ioc-providers npm,ghsa,cve

# Disable IoC scanning
nullvoid /path/to/project --no-ioc
```

### üì¶ **Multi-Layer Cache System**

Enterprise-grade caching with three layers for optimal performance:

- **L1 (Memory Cache)**: Fast in-memory LRU cache for instant access
- **L2 (File Cache)**: Persistent file-based cache in `.nullvoid-cache/` directory
- **L3 (Redis Cache)**: Optional distributed cache for multi-instance deployments

#### Features:
- **Intelligent Promotion**: Hot data automatically promoted from L2 to L1
- **Cache Analytics**: Comprehensive statistics and hit rate tracking
- **Configurable TTL**: Per-layer time-to-live configuration
- **Automatic Cleanup**: Expired entries automatically removed
- **Cache Statistics**: View detailed cache performance metrics

#### Usage:
```bash
# View cache statistics
nullvoid /path/to/project --cache-stats

# Enable Redis cache
nullvoid /path/to/project --enable-redis
```
